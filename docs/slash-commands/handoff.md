# /handoff

目的: 次のエージェント（または将来の自分）が素早く再開できるよう現在の状態をパッケージ化。

含める内容（順番通り）:
1) スコープ/状態: 何をしていたか、完了したこと、保留中のこと、ブロッカー
2) ワーキングツリー: `git status -sb` サマリーとプッシュされていないローカルコミットの有無
3) ブランチ/PR: 現在のブランチ、関連する PR 番号/URL、CI 状態（わかれば）
4) 実行中プロセス: tmux セッション/ペインのリストとアタッチ方法:
   - 例: `tmux attach -t codex-shell` または `tmux capture-pane -p -J -t codex-shell:0.0 -S -200`
   - dev サーバー、テスト、デバッガー、バックグラウンドスクリプトを記載
5) テスト/チェック: 実行したコマンド、結果、まだ実行が必要なもの
6) 次のステップ: 次のエージェントが最初にやるべきことの順序付きリスト
7) リスク/注意点: 不安定なテスト、認証情報、フィーチャーフラグ、脆弱な領域

出力形式: 簡潔な箇条書き。ライブセッション用のコピペ可能な tmux コマンドを含める。

配置場所: グローバルプロンプトは `~/.codex/prompts/handoff.md`。このファイルは編集用のミラー。
